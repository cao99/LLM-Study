### 探讨 1: Token 的本质与切分规则
- **学生初始理解**: 知道 token 是基本单位，但不确定具体怎么切的。
- **教学内容**:
  - Token 不是按"字"切的，而是按"常见模式"切的
  - 模型训练前学习"词表"（vocabulary），尽量用最少的 token 表示文本
  - 英文常见词通常是 1 token = 1 词；中文每个汉字通常是 1-2 tokens
  - 英文罕见长词会被切成多个 token（按词根/前缀切分）
- **实际案例**:
  - `Hello world` = 2 tokens
  - `你好世界` = 4-6 tokens（中文切分成本更高）
  - `antidisestablishmentarianism` = 6-8 tokens
- **理解检查**: 学生正确解释了中文 token 成本更高的原因——词表覆盖率问题，中文词汇组合太多，每个汉字单独成 token 的概率更高。
- **掌握的课题**: Token 的本质、切分规则、中英文 token 成本差异。

---

### 探讨 2: Transformer 架构与注意力机制
- **学生初始理解**: 听说过 Transformer，但不知道具体怎么工作的。
- **教学内容**:
  - 核心思想："看上下文 → 找关联 → 预测下一个"
  - 注意力机制：每个 token 计算与其他 token 的关系权重
  - 位置编码：让每个 token 知道"我在哪里"
  - 多头注意力：同时关注多种类型的关系（代词指代、动词-宾语等）
  - 完整流程：Token化 → 位置编码 → 多头注意力 → 前馈网络 → 预测
- **理解检查**: 学生准确总结——"每个 token 都有自己的位置编码，然后计算注意力权重，通过权重值判断自己和哪个 token 的关系最大。"
- **掌握的课题**: Transformer 核心架构、注意力机制原理、多头注意力的作用。

---

### 探讨 3: 上下文窗口
- **学生初始理解**: 不确定为什么 AI 会有"记忆限制"。
- **教学内容**:
  - 上下文窗口 = 模型一次能"看到"的最大 token 数量
  - 限制原因 1：注意力机制计算复杂度 O(n²)，token 翻倍计算量翻 4 倍
  - 限制原因 2：GPU 显存限制
  - 不同模型的窗口大小：GPT-3.5 (4k), GPT-4 (8k-128k), Claude 3 (200k)
  - 对话时的实际发送：系统提示 + 所有历史对话 + 新问题
  - 超出窗口时，最早的内容会被"挤出"
- **理解检查**: 学生正确回答了超出上下文窗口的后果——模型会截断后面的内容，只处理前 N 个 tokens。
- **掌握的课题**: 上下文窗口的概念、限制原因、实际影响、对话"遗忘"的原理。
