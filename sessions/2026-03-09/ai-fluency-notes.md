## 知识域 4：Agentic Workflow 与智能体

### 探讨 1: 传统自动化 vs AI Agent
- **学生初始理解**: 有丰富的 Jenkins Pipeline、Ansible Playbook、cron 使用经验。
- **教学内容**: 传统自动化 = 确定性控制流（人写死所有分支），AI Agent = LLM 实时决策 + 动态循环。
- **理解检查**: 学生正确回答了传统自动化"只能按预先写好的 when/catch 逻辑处理"。
- **掌握的课题**: 传统自动化与 AI Agent 的本质区别（高置信度）。

### 探讨 2: Agent 核心工作循环与三大组件
- **教学内容**: 观察-思考-行动循环（ReAct）、三大组件（LLM/Tools/Memory）、微服务架构类比。
- **理解检查**: 学生理解了 IDE AI 助手就是 Agent 实例、Tools 决定 Agent 能力边界。
- **掌握的课题**: Agent 工作循环、三大核心组件（高置信度）。

### 探讨 3: Function Calling 与 Agent 风险
- **教学内容**: Function Calling 机制、LLM 本质仍是 Token 预测、幻觉工具调用和危险操作风险、Human-in-the-Loop。
- **掌握的课题**: Function Calling 机制、Agent 风险与 HITL（高置信度）。

### 探讨 4: 实操体验 — 扣子(Coze)平台
- **实操内容**: 在 coze.cn 体验对话式 AI，输入 K8s CrashLoopBackOff 排查问题。
- **教学重点**: 复盘分析"聊天机器人"vs"真正 Agent"的区别。
- **理解检查**: 学生准确回答"变成真正 Agent 还需要记忆和工具"。
- **掌握的课题**: 聊天机器人与 Agent 的本质区别（高置信度）。

### 插曲：低代码 Agent 平台全景介绍
- 对比了扣子(Coze)、Dify、LangChain、FastGPT、百度千帆、阿里百炼的定位和适用场景。
- 核心结论：这些平台是"Agent 的乐高积木"，帮用户低成本组装 LLM + Tools + Memory。

## 知识域 2：日常交互与生产力提效

### 探讨 1: CREF Prompt 框架
- **教学内容**: Prompt 四要素 — Context（上下文）、Role（角色）、Explicit（明确要求）、Feedback（约束）。
- **实操练习**: 学生用 CREF 框架编写了 K8s 集群告警规则的 Prompt，评分 9/10（上下文精确、角色到位、监控维度清晰、约束合理），并获得了高质量的告警规则回答。
- **掌握的课题**: CREF Prompt 框架（高置信度）。

### 探讨 2: 思维链 CoT
- **教学内容**: 三种方式——魔法咒语、指定推理步骤、Few-Shot 示例。核心原理：分步推理降低每步 Token 预测难度。
- **实操练习**: 学生尝试写 KubeSphere 升级评估的 CoT Prompt，使用了方式1但缺少上下文。引导学生理解"CREF + CoT 缺一不可"。
- **掌握的课题**: CoT 原理与用法、CREF+CoT 组合使用（高置信度）。

### 探讨 3: 角色扮演
- **教学内容**: 三种高阶玩法——对立角色审查、专家面板、模拟用户。
- **掌握的课题**: 角色扮演的高阶用法（高置信度）。

### 探讨 4: 格式约束
- **教学内容**: 表格/JSON/Markdown模板/长度限制。格式约束的好处：可直接使用、减少废话、方便对比、可编程处理。
- **掌握的课题**: 格式约束技巧（高置信度）。

### 综合理解检查
- 学生正确回答了编写月度巡检报告 Prompt 的思路：先用 CREF 框架、再指定角色、最后格式约束。
- 小改进空间：可补充 CoT 来指定巡检的思考步骤。

## 📋 知识域完整技能树

### 知识域 4：Agentic Workflow 与智能体

```
Agentic Workflow 与智能体
│
├── 1. 传统自动化 vs AI Agent
│   ├── 传统自动化 = 确定性控制流（人写死所有分支）
│   └── AI Agent = LLM 实时决策 + 动态循环
│
├── 2. Agent 核心工作循环（ReAct 模式）
│   ├── 观察 (Observe) ← 接收任务 / 获取当前状态
│   ├── 思考 (Think/Plan) ← LLM 分析情况，决定下一步
│   ├── 行动 (Act) ← 调用工具执行操作
│   └── 反馈 (Observe again) ← 检查结果，决定继续或完成
│
├── 3. Agent 三大核心组件
│   ├── 🧠 LLM（大脑）：理解、推理、决策
│   ├── 🔧 Tools（工具/手脚）：执行外部操作（Shell、API、DB）
│   └── 💾 Memory（记忆）：短期（对话上下文）+ 长期（知识库）
│
├── 4. Function Calling 机制
│   ├── LLM 输出结构化 JSON（工具名 + 参数）
│   ├── 编排框架执行工具，结果喂回 LLM
│   └── 本质仍是 Token 预测（不是"真的理解"在调用工具）
│
├── 5. Agent 风险与防护
│   ├── 幻觉工具调用（编造不存在的工具或错误参数）
│   ├── 危险操作（LLM 不理解后果，可能预测出破坏性命令）
│   └── 铁律：高危操作必须 Human-in-the-Loop（人类审批）
│
└── 6. 低代码 Agent 平台
    ├── 扣子 (Coze)：个人 AI 助手，零代码建 Bot
    ├── Dify：企业级，可私有化部署，完整 RAG 管线
    ├── LangChain：开发者框架，需写 Python
    └── FastGPT：轻量级，专注知识库问答
```
